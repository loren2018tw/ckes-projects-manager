## Context

目前專案使用 Google Drive API 作為資料儲存後端，各模組資料以 JSON 格式儲存在使用者 Google Drive 的應用程式資料夾中。既有模組（專案管理、聯絡人管理）皆遵循此模式。

本模組「請購單管理」為全新的獨立模組，不修改既有功能。需與專案管理模組建立關聯：請購單的「使用專款」欄位下拉選取未結案專案。

## Goals / Non-Goals

**Goals:**

- 新增請購單管理獨立頁面與路由
- 請購單 CRUD（新增、編輯、刪除、複製）
- 請購項目 CRUD（新增、編輯、刪除）
- 請購單與請購項目以對話框方式編輯
- 請購單清單點選後，下方顯示該單的請購項目
- 專款欄位下拉連結未結案專案
- 新增左側選單導航項目

**Non-Goals:**

- 不做列印功能（按鈕預留但不實作）
- 不做權限控管（沿用既有 Google OAuth 的使用者層級）
- 不做請購單匯出/匯入

## Decisions

### 資料儲存方式：沿用 Google Drive JSON 檔案

- 使用既有 `useDriveStorage.js` composable 存取資料
- 請購單主檔儲存為 `purchase-requests.json`，陣列格式
- 請購項目依附於請購單內（作為請購單物件的 `items` 陣列欄位），同一檔案儲存
- **原因**：請購項目與請購單的生命週期一致，不需獨立檔案；減少 API 請求次數

### 專款資料取得方式

- 透過既有 `useDriveStorage.js` 讀取 `projects.json`，過濾出 `status !== 'closed'` 的專案
- 下拉選單顯示專案名稱，選取後儲存專案 ID
- **原因**：不引入新依賴，Googel Drive 已有專案清單資料

### 複製請購單實作方式

- 讀取原始請購單所有欄位，將 `date` 改為當日日期，`id` 重新產生
- 複製 `items` 陣列（為每個 item 產生新 id）
- 寫入檔案

### 路由設計

- 路徑：`/purchase-requests`
- 單一頁面元件：`PurchaseRequestManagement.vue`（上半部請購單清單，下半部請購項目清單）

### 對話框編輯

- 新增/編輯請購單：使用 Quasar `QDialog` 內嵌表單
- 新增/編輯請購項目：使用 Quasar `QDialog` 內嵌表單
- 刪除前顯示確認對話框

## Risks / Trade-offs

- **效能風險**：請購項目與請購單儲存在同一檔案，若某筆請購單含大量項目，讀寫該筆時可能較慢 → 待實際使用後若出現效能問題，再拆分為獨立檔案
- **資料一致性**：專案結案後，既有請購單的「使用專款」仍保留原專案名稱（不連動更新） → 本模組為唯讀引用，不影響既有資料
