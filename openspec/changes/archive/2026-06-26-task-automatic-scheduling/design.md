## Context

目前任務管理系統支援三種相依模式（FS、SS、FF），但日期完全由使用者手動設定。當使用者建立或變更相依關係時，系統不自動調整日期；前置任務延遲時，後續任務也不會連動更新。甘特圖雖有基礎的相依箭頭繪製，但該邏輯僅用於顯示，未回寫至任務資料。

欲解決此問題，需在不變更資料結構的前提下，新增自動排程引擎，於特定觸發點（新增/編輯任務、變更狀態、變更前置任務）自動計算並更新任務日期。

## Goals / Non-Goals

**Goals:**

- 實作自動排程引擎，根據相依模式計算任務日期
- FS（完成-開始）：任務開始日期 = 前置任務結束日期 + 1 天
- SS（開始-開始）：任務開始日期 = 前置任務開始日期（同步開始）
- FF（完成-完成）：任務截止日期 = 前置任務截止日期（同步完成）
- 前置任務日期變更時，連鎖更新所有下游相依任務
- 所有排程邏輯僅操作現有欄位（startDate、deadline、completedDate、status）
- 提供衝突偵測與提示（排程無法滿足相依條件時）
- 排程發生時在 UI 顯示調整資訊

**Non-Goals:**

- 不新增資料結構欄位
- 不實作資源平衡或工作量分配
- 不支援 SF（開始-完成）相依模式
- 不修改 Google Drive 儲存格式
- 不修改看板/列表/甘特圖的渲染邏輯

## Decisions

### 1. 採用 composable 模式封裝排程引擎

選擇將排程邏輯抽取為獨立 composable `useScheduleEngine`，而非直接寫入 taskStore。

- **理由**：排程邏輯為純計算（pure computation），與儲存/狀態分離更易測試。taskStore 負責資料存取，scheduleEngine 負責排程計算。
- **替代方案**：直接寫入 taskStore。但 taskStore 已 218 行，加入排程邏輯會使其職責過重。

### 2. 排程引擎回傳建議變更而非直接修改

scheduleEngine 接收任務清單與觸發任務 ID，回傳一組「建議的日期變更」，由呼叫方決定是否套用。

- **理由**：讓 UI 層能先顯示排程調整建議，使用者確認後再儲存。避免自動修改造成使用者困惑。
- **替代方案**：直接在 engine 內修改陣列。但無法提供「預覽」功能，使用者體驗較差。

### 3. 使用 BFS 進行連鎖更新

當某任務日期變更時，用 BFS 遍歷所有依賴它的任務，逐層計算。

- **理由**：相依鏈可能很長（A→B→C→D），BFS 確保每一層都在前一層計算完成後才處理，避免順序問題。
- **替代方案**：遞迴 DFS。但深度過大時可能導致堆疊溢出，BFS 更安全。

### 4. 排程觸發點統一在 taskStore 的 update/add 方法

在 taskStore.update() 和 taskStore.add() 完成後，自動呼叫 scheduleEngine 進行排程。

- **理由**：無論從 UI 何處修改任務（看板拖曳、編輯對話框、程式化呼叫），都會經過 store 的這些方法，確保排程邏輯一致。
- **替代方案**：在 UI 層各自觸發。容易遺漏，造成行為不一致。

### 5. 甘特圖暫不連動自動排程

甘特圖本身只負責顯示，目前的 `_ganttStartX` 等 computed 中已有相依調整顯示邏輯，但那是唯讀的。自動排程會實際修改 startDate/deadline，甘特圖自然會反映變更。

### 6. 衝突處理策略

當排程導致某任務的 startDate > deadline 時：

- 若 deadline 為手動設定 → 保留 deadline，調整 startDate（彈性開始）
- 若 deadline 也被相依條件鎖定（FF 模式） → 提示使用者存在排程衝突，由使用者決定調整何者
- 簡化處理：優先將 deadline 調整為 startDate（與現有 validateTask 行為一致）

## Risks / Trade-offs

- **連鎖更新的儲存次數**：每次連鎖更新都寫入 Google Drive 可能產生大量 API 呼叫 → 採用批次儲存，整個排程計算完後一次寫入
- **使用者覆寫自動排程**：使用者可能手動修改已被排程調整的日期 → 設計為「排程建議優先，但使用者可手動覆寫」，覆寫時顯示提示
- **無窮迴圈**：相依關係形成循環時，排程計算會無限循環 → 在排程前先執行循環偵測（重用現有 hasCycle），有循環時跳出排程
- **效能**：大量任務（>200）時連鎖更新可能變慢 → scheduleEngine 使用 BFS + 快取計算結果，避免重複運算
