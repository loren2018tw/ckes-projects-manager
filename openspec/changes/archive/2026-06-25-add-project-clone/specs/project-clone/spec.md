## ADDED Requirements

### Requirement: 專案列表顯示複製按鈕

專案列表每一筆專案右側需顯示「複製」按鈕，供使用者快速發起複製流程。

#### Scenario: 專案列表右側顯示複製按鈕

- **WHEN** 使用者進入專案列表頁面
- **THEN** 每筆專案右側（編輯、刪除按鈕旁）顯示「複製」按鈕

### Requirement: 開啟複製對話框

使用者點擊複製按鈕後，系統顯示複製對話框，供輸入新專案名稱及選擇是否複製資源檔。

#### Scenario: 點擊複製按鈕顯示對話框

- **WHEN** 使用者點擊任一專案的「複製」按鈕
- **THEN** 系統彈出對話框，標題顯示「複製專案」，對話框包含新專案名稱輸入欄（預設值為「原始專案名稱 - 複製」）與「複製資源檔案」勾選框（預設勾選）

#### Scenario: 點擊取消關閉對話框

- **WHEN** 使用者在複製對話框中點擊「取消」按鈕或點擊對話框外部
- **THEN** 對話框關閉，不執行任何複製動作

### Requirement: 驗證新專案名稱

系統需驗證新專案名稱不可為空白且不可與現有專案名稱重複。

#### Scenario: 名稱為空白時無法複製

- **WHEN** 使用者將新專案名稱留空，點擊「複製」按鈕
- **THEN** 系統顯示提示「專案名稱不可為空白」，不執行複製

#### Scenario: 名稱與現有專案重複時無法複製

- **WHEN** 使用者輸入的名稱與現有專案名稱重複，點擊「複製」按鈕
- **THEN** 系統顯示提示「此專案名稱已存在」，不執行複製

### Requirement: 任務複製與清理

複製專案時，系統須完整複製所有任務，保留任務間的相依關係，但清空開始日期、截止日期、負責人欄位。

#### Scenario: 成功複製任務並保留相依

- **WHEN** 使用者輸入新專案名稱並確認複製
- **THEN** 系統將原始專案的所有任務複製到新專案，任務間的相依關係完全保留，每個任務的開始日期、截止日期、負責人被清空為空白，任務狀態設為「未開始」

#### Scenario: 無任務時不複製任務

- **WHEN** 原始專案沒有任何任務
- **THEN** 新專案的任務清單為空

### Requirement: 選擇性複製資源檔案

使用者可在複製對話框中選擇是否一併複製資源檔案。

#### Scenario: 勾選複製資源檔

- **WHEN** 使用者在複製對話框中勾選「複製資源檔案」並確認複製
- **THEN** 系統將原始專案資源管理中的所有檔案（含分類資料夾結構）複製到新專案的資源管理中

#### Scenario: 未勾選複製資源檔

- **WHEN** 使用者在複製對話框中取消勾選「複製資源檔案」並確認複製
- **THEN** 新專案的資源管理為空的初始狀態

#### Scenario: 資源檔案複製完成訊息

- **WHEN** 資源檔案複製完成
- **THEN** 系統顯示成功訊息「專案複製完成」

### Requirement: 複製中顯示進度

複製過程中系統須顯示載入狀態或進度指示，避免使用者重複操作。

#### Scenario: 複製中顯示載入狀態

- **WHEN** 使用者確認複製後，系統正在執行複製
- **THEN** 對話框顯示載入狀態或進度條，複製按鈕變為不可用狀態，防止重複點擊

#### Scenario: 複製完成後導向新專案

- **WHEN** 複製完成
- **THEN** 對話框關閉，系統導航至新專案的專案管理頁面

### Requirement: 複製中錯誤處理

複製過程中若發生錯誤，系統需提示使用者並部分還原。

#### Scenario: 複製過程中發生錯誤

- **WHEN** 複製過程中 Google Drive API 回傳錯誤（如配額超限、網路中斷）
- **THEN** 系統顯示錯誤訊息「複製失敗：{{錯誤原因}}」，已建立的資料應嘗試清除或提示使用者手動處理
