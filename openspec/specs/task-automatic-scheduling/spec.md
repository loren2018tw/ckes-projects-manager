# task-automatic-scheduling Specification

## Purpose

自動排程引擎，根據任務相依關係自動計算與調整任務日期，支援連鎖更新與衝突處理。不修改現有資料結構。

## Requirements

### Requirement: FS 相依模式自動排程

任務設定為 FS（完成-開始）相依模式時，系統自動計算任務開始日期。

#### Scenario: FS 新增任務時自動計算開始日期

- **WHEN** 使用者新增任務並選擇前置任務（相依模式為 FS），前置任務有截止日期
- **THEN** 系統自動將新任務的開始日期設為「前置任務截止日期 + 1 天」

#### Scenario: FS 前置任務已完成時使用完成日期計算

- **WHEN** 使用者新增任務並選擇前置任務（相依模式為 FS），前置任務狀態為「已完成」
- **THEN** 系統自動將新任務的開始日期設為「前置任務完成日期 + 1 天」

#### Scenario: FS 前置任務無截止日期且未完成

- **WHEN** 使用者新增任務並選擇前置任務（相依模式為 FS），前置任務無截止日期且未完成
- **THEN** 系統不自動設定開始日期，由使用者手動輸入

#### Scenario: FS 前置任務日期變更時連鎖更新

- **WHEN** 前置任務的截止日期或完成日期變更
- **THEN** 系統自動連鎖更新所有 FS 相依任務的開始日期

### Requirement: SS 相依模式自動排程

任務設定為 SS（開始-開始）相依模式時，系統自動計算任務開始日期。

#### Scenario: SS 新增任務時自動設定開始日期

- **WHEN** 使用者新增任務並選擇前置任務（相依模式為 SS），前置任務有開始日期
- **THEN** 系統將新任務的開始日期設為與前置任務開始日期相同

#### Scenario: SS 前置任務日期變更時連鎖更新

- **WHEN** 前置任務的開始日期變更
- **THEN** 系統自動將所有 SS 相依任務的開始日期同步更新為相同日期

### Requirement: FF 相依模式自動排程

任務設定為 FF（完成-完成）相依模式時，系統自動計算任務截止日期。

#### Scenario: FF 新增任務時自動設定截止日期

- **WHEN** 使用者新增任務並選擇前置任務（相依模式為 FF），前置任務有截止日期
- **THEN** 系統將新任務的截止日期設為與前置任務截止日期相同

#### Scenario: FF 前置任務日期變更時連鎖更新

- **WHEN** 前置任務的截止日期變更
- **THEN** 系統自動將所有 FF 相依任務的截止日期同步更新為相同日期

### Requirement: 連鎖排程更新

當一個任務的日期因編輯或狀態變更而改變時，所有直接或間接依賴該任務的任務都應自動更新。

#### Scenario: 連鎖更新鏈

- **WHEN** 任務 A（前置任務）的截止日期變更，任務 B 以 FS 相依於 A，任務 C 以 FS 相依於 B
- **THEN** 系統依序計算：任務 B 開始日期 = A 新截止日 + 1，任務 C 開始日期 = B 新截止日 + 1

#### Scenario: 狀態變更為已完成觸發連鎖更新

- **WHEN** 使用者將前置任務狀態變更為「已完成」，設定完成日期
- **THEN** 系統以完成日期為基準，連鎖更新所有相依任務的日期

#### Scenario: 前置任務解除時不觸發排程

- **WHEN** 使用者移除任務的相依關係（清空前置任務）
- **THEN** 系統不清除該任務的日期，保留目前設定值

### Requirement: 排程衝突處理

當自動排程導致任務的開始日期晚於截止日期時，系統需處理衝突。

#### Scenario: FS 排程導致開始日期晚於截止日期時自動調整

- **WHEN** 前置任務截止日期較晚，使相依任務的開始日期超過其既有的截止日期
- **THEN** 系統將該任務的截止日期調整為與開始日期相同，並顯示排程調整通知

#### Scenario: 排程衝突通知

- **WHEN** 自動排程調整了任何任務的日期
- **THEN** 系統在畫面顯示通知，列出被調整的任務名稱與調整內容

### Requirement: 排程計算流程

系統需明確定義排程引擎的計算流程與觸發時機。

#### Scenario: 新增任務後自動排程

- **WHEN** 使用者新增任務並設定相依關係，儲存成功後
- **THEN** 系統自動執行排程引擎，更新所有受影響任務的日期，並一次寫入 Google Drive

#### Scenario: 編輯任務後自動排程

- **WHEN** 使用者編輯任務（修改日期、狀態、前置任務、相依模式任一項），儲存成功後
- **THEN** 系統自動執行排程引擎，更新所有受影響任務的日期

#### Scenario: 循環相依時不執行排程

- **WHEN** 任務設定導致相依循環（hasCycle 回傳 true）
- **THEN** 系統拒絕儲存，不執行排程

### Requirement: 排程引擎 API

排程引擎以 composable 形式提供，供 taskStore 與 UI 層使用。

#### Scenario: scheduleEngine 提供排程方法

- **WHEN** taskStore 呼叫 scheduleEngine.schedule(tasks, triggerTaskId)
- **THEN** 回傳調整後的任務清單，包含每個被調整任務的調整說明

#### Scenario: scheduleEngine 不修改原始陣列

- **WHEN** taskStore 呼叫 scheduleEngine.schedule(tasks, triggerTaskId)
- **THEN** 原始 tasks 陣列不被修改，回傳新的任務副本陣列
